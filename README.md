# Azure ETL Customer Product Dashboard

## 🚀 Project Overview
This project demonstrates an end-to-end ETL pipeline built using Azure Databricks and SQL.

The pipeline processes customer, product, and date data and creates a unified analytical view for dashboard reporting.

## 🔧 Tech Stack
- Azure Databricks
- SQL
- Python
- Data Modeling (Fact & Dimension Tables)

## 📊 Features
- Data transformation using SQL
- Fact and Dimension data modeling
- Analytical view creation
- Dashboard for trend and distribution analysis

## 📊 Insights

- The dataset contains around **10K total interactions**, with approximately **4.1K unique customers and products**, indicating a well-distributed dataset.

- **Category Trends**:
  - Apparel and Home & Kitchen categories show consistently higher activity across months.
  - Category distribution remains stable with slight variation between August, September, and October.

- **Top Brands**:
  - Brands like *CasaNova*, *CookMate*, and *AquaPure* have the highest interaction counts.
  - Brand performance is relatively balanced, with no extreme outliers.

- **Customer Activity Trend**:
  - Daily activity remains fairly consistent over time with minor fluctuations.
  - No sharp spikes or drops, indicating stable engagement.

- **Regional / Distribution Insight**:
  - Customer interactions are spread across multiple regions, showing no heavy dependency on a single region.

- **Weekend vs Weekday Behavior**:
  - Activity is slightly higher on weekends compared to weekdays, indicating increased engagement during non-working days.

- **Activity Heatmap**:
  - Higher activity is observed on **Friday and Sunday**, suggesting peak engagement periods.
  - Mid-week activity remains moderate and consistent.
